# 🧾 אוטומציית הפקת קבלות — חשבונית ירוקה (morning)

אפליקציה מקומית להעלאת אקסל לקוחות והפקת קבלות אוטומטית דרך ה-API של morning (חשבונית ירוקה), כולל שליחת המסמך למייל של כל לקוח.

מיועדת כרגע לעמותה (מפיקה **קבלה**, סוג 400), אך ניתן לעבור בדף ההגדרות ל**קבלה על תרומה מוכרת למס** (405, סעיף 46), לחשבונית מס-קבלה (320) או לחשבונית מס (305).

## התקנה

```bash
cd greeninvoice-automation
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## הגדרת מפתחות (מאובטח)

1. צור מפתח API בחשבונית ירוקה: **אזור אישי > כלים למפתחים > יצירת מפתח** (דורש מסלול Best ומעלה). מומלץ ליצור גם מפתח לסביבת Sandbox.
2. העתק את קובץ הדוגמה ומלא ערכים:

```bash
cp .env.example .env
```

3. ערוך את `.env` והזן את המפתחות. **הקובץ לא נשמר ב-git** (מוגדר ב-`.gitignore`) ולא מוצג בממשק — כך הסודות נשארים רק במחשב שלך.

## הרצה

```bash
streamlit run app.py
```

ייפתח דפדפן עם שני מסכים:

- **מסך ראשי** — העלאת האקסל, תצוגה מקדימה (Dry Run) של מה יופק/ידולג, אישור מפורש, הרצה, ודוח סיכום עם קישורים למסמכים והורדת CSV.
- **⚙️ Settings** — סביבה (Sandbox/ייצור), סוג מסמך, הגדרות מע"מ, תיאור השירות (עם משתנים `{event}` `{order_id}` `{name}` `{date}`), הערות, שליחת מייל, דילוג על 0 ₪, מקור התאריך, ומיפוי צורות תשלום.

## מבנה האקסל הנדרש

| מספר הזמנה | תאריך | שם פרטי | שם משפחה | טלפון | מייל | אירוע | סכום | סכום לפי מטבע דיפולטיבי | צורת תשלום |
|---|---|---|---|---|---|---|---|---|---|

- הסכום נלקח מעמודת "סכום לפי מטבע דיפולטיבי" (או "סכום" אם היא חסרה); תווים כמו `¦` מנוקים אוטומטית.
- תאריך בפורמט `02.07.2026 18:27` (נתמכים גם פורמטים נפוצים נוספים).

## מנגנוני בטיחות

- **Sandbox כברירת מחדל** — עברו לייצור רק אחרי בדיקה מוצלחת.
- **Dry Run מובנה** — כל קובץ מוצג קודם כתוכנית פעולה; ההפקה דורשת סימון אישור מפורש.
- **מניעת כפילויות** — כל מספר הזמנה שהופק נרשם ב-`data/processed_<env>.json`; הרצה חוזרת של אותו קובץ תדלג עליו. (למחיקת ההיסטוריה — מחקו את הקובץ.)
- **חריגות לא עוצרות את הריצה** — שורה עם מייל שגוי או שגיאת API מדווחת בדוח וממשיכים הלאה.

## שליחת המייל ללקוח

המסמך נשלח על ידי מערכת חשבונית ירוקה עצמה לכתובת שבשדה `client.emails`. ודאו שבהגדרות העסק במורנינג מופעלת שליחת מסמכים במייל. ניתן לכבות שליחה בדף ההגדרות (המסמך יופק בלי מייל).

## ⚠️ לאימות בריצת ה-Sandbox הראשונה

הקוד נכתב לפי התיעוד הרשמי (developers.morning.co), אך כמה פרטים כדאי לאמת מול החשבון שלכם בפועל:

1. **כתובת ה-Sandbox** — ברירת המחדל היא `https://sandbox.d.greeninvoice.co.il/api/v1`. אם השתנתה, עדכנו את `GI_SANDBOX_BASE_URL` ב-`.env`. (כתובת הייצור החדשה `https://api.greeninvoice.co.il/api` מחייבת `grant_type: client_credentials` — כבר מיושם בקוד.)
2. **קודי צורות תשלום** — המיפוי (3=אשראי וכו') ניתן לעריכה בדף ההגדרות אם קוד כלשהו נדחה.
3. **הגדרות מע"מ לעמותה** — ברירת המחדל לשורת הכנסה היא "פטור" (2). ודאו שזה תואם את הגדרת העסק.
4. **קבלה על תרומה (405)** — דורשת שהעסק במורנינג מוגדר כמוסד עם אישור סעיף 46.
