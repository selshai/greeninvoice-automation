# 🧾 אוטומציית הפקת קבלות — חשבונית ירוקה (morning)

אפליקציה מקומית (Streamlit) להעלאת קובץ אקסל לקוחות והפקת קבלות אוטומטית דרך ה-API של **morning (חשבונית ירוקה)**, כולל שליחת המסמך למייל של כל לקוח.

מיועדת כברירת מחדל לעמותה (מפיקה **קבלה**, סוג 400), אך ניתן לעבור בדף ההגדרות ל**קבלה על תרומה מוכרת למס** (405, סעיף 46), ל**חשבונית מס-קבלה** (320) או ל**חשבונית מס** (305).

---

## 🚀 הפעלה מהירה (מומלץ)

מתקין הכול אוטומטית (כולל Python אם חסר), יוצר סביבה, מתקין ספריות, מפעיל את השרת ופותח דפדפן — עם פס התקדמות ושלבים.

- **Windows** — לחיצה כפולה על **`run.bat`**
- **macOS** — לחיצה כפולה על **`run.command`** (בפעם הראשונה בלבד, הריצו בטרמינל: `chmod +x run.command`)

בסיום ייפתח דפדפן בכתובת <http://localhost:8501>.

### הפעלה ידנית (חלופה)

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

## 🔑 הגדרת מפתחות API

יש שתי דרכים — שתיהן שומרות את המפתחות בקובץ `.env` המקומי בלבד (**לא נשמר ב-git**):

1. **דרך הממשק (מומלץ):** בדף ⚙️ Settings → *עדכון מפתחות API* → הדביקו Key ו-Secret → *שמירת מפתחות*. כפתור *בדיקת חיבור* מוודא שהמפתחות תקינים.
2. **ידנית:** העתיקו `.env.example` ל-`.env` ומלאו את הערכים.

יצירת מפתח: בחשבונית ירוקה → **אזור אישי → כלים למפתחים → יצירת מפתח** (דורש מסלול מתאים). מומלץ ליצור מפתח נפרד לסביבת **Sandbox** (מערכת נפרדת עם התחברות נפרדת).

## 🧭 זרימת העבודה

האפליקציה בנויה משני שלבים ברורים:

**שלב 1 — בחירה ותצוגה מקדימה**
- העלאת האקסל (נשמר בזיכרון גם במעבר בין דפים).
- טבלת *Dry Run*: מה יופק / ידולג / חריגה, עם סיבה לכל שורה.
- **לחיצה על שורה** מציגה תצוגה מקדימה של המסמך כפי שיישלח (כולל ה-JSON המדויק).

**שלב 2 — אישור והרצה**
- סיכום כמות המסמכים והסביבה, אזהרת ייצור, סימון אישור מפורש.
- פס התקדמות בזמן ההפקה; התוצאות נשמרות במסך ולא נעלמות ברענון.
- דוח סיכום עם קישורים למסמכים והורדת CSV.

## ⚙️ הגדרות (דף Settings)

- **סביבה** — Sandbox / ייצור, כולל הזנת מפתחות ובדיקת חיבור.
- **סוג מסמך ומע"מ** — סוג המסמך, vatType ברמת המסמך וברמת שורת ההכנסה.
- **תיאור והערות** — תבניות עם המשתנים `{event}` `{order_id}` `{name}` `{date}` `{description}`.
  - ℹ️ בקבלה (400) תיאור השירות אינו מוצג כשורה — השתמשו ב-`{description}` בתבנית ההערות כדי שיופיע על המסמך.
- **התנהגות** — שליחת מייל, דילוג על 0 ₪, הוספת לקוח ל-CRM, מקור התאריך (מהאקסל / היום).
- **הגבלת מספר מסמכים** — הפקה של N השורות הראשונות בלבד (נוח לבדיקה).
- **ניסיון חוזר** — עיבוד רק הזמנות שנכשלו בעבר; כשכבוי, הזמנות שנכשלו מדולגות אוטומטית בהעלאה חוזרת.
- **מיפוי עמודות האקסל** — שם הכותרת בקובץ שלכם לכל שדה (לתמיכה בקבצים בפורמטים שונים), עם כפתור **🧪 בדיקת מיפוי** שמראה אילו כותרות תואמו לפני הרצה.
- **מיפוי צורות תשלום** — טקסט צורת התשלום באקסל → קוד תשלום ב-API.

## 📄 מבנה האקסל

ברירת המחדל לכותרות (ניתן לשנות ב*מיפוי עמודות*):

| מספר הזמנה\* | תאריך | שם פרטי | שם משפחה | טלפון | מייל\* | אירוע | סכום\* | צורת תשלום |
|---|---|---|---|---|---|---|---|---|

\* שדות חובה.

- הסכום מנוקה מתווים כמו `₪` / `¦` / פסיקים אוטומטית.
- כותרות עם רווח קשיח (NBSP) מזוהות אוטומטית.
- תאריך בפורמט `02.07.2026 18:27` ופורמטים נפוצים נוספים.

## 🛡️ מנגנוני בטיחות

- **Dry Run + אישור מפורש** — שום דבר לא מופק בלי שראיתם תצוגה מקדימה ואישרתם.
- **מניעת כפילויות** — כל מספר הזמנה שהופק נרשם ב-`data/processed_<env>.json`; הרצה חוזרת מדלגת עליו.
- **לוג הרצות מתמשך** — כל ניסיון (הצלחה/כישלון) נשמר ב-`data/runlog_<env>.jsonl` וניתן לצפייה/הורדה מהמסך הראשי.
- **חריגות לא עוצרות את הריצה** — שורה עם שגיאה מדווחת וממשיכים; נכשלות אינן נכנסות לרישום הכפילויות וניתן לנסותן שוב.
- **הפרדת סביבות** — היסטוריה ולוג נפרדים ל-Sandbox ולייצור.

> ⚠️ **פרטיות:** התיקייה `data/` (שמות, מיילים, מספרי הזמנה) וקובץ `.env` (מפתחות) אינם נשמרים ב-git. אל תמחקו את `data/processed_production.json` — זהו הרישום היחיד של מה שכבר הופק.
